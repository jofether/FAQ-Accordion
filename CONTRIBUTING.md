# Contributing Guide

Thank you for your interest in contributing to our Synthetic Dataset project! We welcome contributions of all kinds.

## How to Contribute

### Report Bugs
If you find a bug, please:
1. Check if it has already been reported in the issues
2. Create a new issue with a clear title and description
3. Include steps to reproduce and expected behavior
4. Add screenshots if relevant

### Submit Bug Fixes
1. Fork the repository
2. Create a feature branch (`git checkout -b fix/bug-description`)
3. Make your changes
4. Write clear commit messages
5. Push to your fork and create a pull request

### Add New FAQs
We're always looking to expand our FAQ content. To suggest new questions and answers:
1. Create an issue with the label `enhancement`
2. Provide the question and detailed answer
3. Include the category (technical, dataset, usage)
4. Our team will review and merge

### Improve Documentation
Help us improve our documentation:
- Fix typos or unclear explanations
- Add examples or use cases
- Improve UI/UX descriptions
- Update technical accuracy

### Expand the Dataset
Contribute new synthetic website designs:
1. Follow our dataset format guidelines
2. Include source code and screenshots
3. Provide bug annotations if applicable
4. Submit via pull request with detailed description

## Pull Request Process

1. Update the README.md with any new features or changes
2. Add comments for complex logic
3. Ensure all tests pass (if applicable)
4. Keep commit history clean and meaningful
5. Use clear commit messages

## Code Style

- Follow React best practices
- Use meaningful variable and function names
- Keep components focused and reusable
- Add comments for non-obvious logic
- Format with 2-space indentation

## Dataset Contribution Guidelines

When contributing new synthetic websites:
- Use React + Tailwind CSS
- Include responsive designs
- Provide multiple resolution screenshots
- Document the code structure
- List any intentional bugs in annotations
- Follow existing naming conventions

## Testing

Before submitting:
- Test the accordion functionality
- Verify responsive design on mobile/desktop
- Check category filtering works correctly
- Ensure animations are smooth
- Test keyboard navigation

## Getting Help

- Join our discussions for general questions
- Check existing issues for known problems
- Read the main README for setup instructions
- Contact the team for clarification
